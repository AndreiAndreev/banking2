Api for using emailhuter with chrome-extension and prospecting



Class EmailHunterController in package com.controllers has a method, which take 3 parameters from chrome-extension, like domain, first name, last name.


Class EmailHunterController in package com.controllers has a method, which take 3 parameters from chrome-extension, like domain, first name, last name.
Chrome extension should send ajax request to

site_adress/emailhunter/process?domain=DOMAIN_NAME&first_name=FIRST_NAME&last_name=LAST_NAME&position=POSITION&company=COMPANY&api_key=API_KEY
This method return json file with result, which chrome-extension can parse.


For verify email use: site_adress/emailhunter/verification?email=EMAIL


Use

site_adress/emailhunter/process/currentUser
to get json with user, which contains first name, last name, api key, available searches, plan name and plan available searches


To get user lists

site_adress/api/getLists?api_key=API_KEY


To save use

site_adress/emailhunter/save?email=EMAIL&first_name=FIRST_NAME&last_name=LAST_NAME&position=POSITION&company=COMPANY&link=LINK&relevance=RELEVANCE&user_ulr=USER_URL&list_name=LIST_NAME&api_key=API_KEY


Now you should use port 8443. This is port for https connection
